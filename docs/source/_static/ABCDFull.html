A Demonstration of Workspace Building and Analysis in xRooFit¶

This tutorial is splits into two parts:

  • Part I - Workspace (model and dataset) Building: Here you will learn about the structure of statistical models in HEP, including the terminology used to describe them.
  • Part II - Statistical Analysis: You will use the workspace you have built in Part I to perform statistical analysis techniques such as running fits, generating datasets, and computing confidence limits.

Today we will perform a statistical analysis for a hypothetical search/measurement of a signal in some signal region (region A). Let us suppose we have designed some event selection for our signal region and come up with an observable, "some obs", that we will fill into a histogram. We produce this histogram from our signal MC sample, along with a histogram for some minor irreducible backgrounds using background MC samples. Both of these MC histograms have MC-statistical uncertainties on them. We also have produced that background histogram with a systematic variation applied, representing some experimental or theoretical uncertainty. Suppose further that we have concluded that our main background should be data driven: we have designed a control region (region B) which is the same selection as our signal region except one of the selection cuts is inverted; the inverted cut gives a region rich in our dominant background. We plan to make a histogram of the observed events in this control region and apply a transfer factor to it before using it as the data-driven background prediction in the signal region. We plan to measure this transfer factor from the ratio between two further control regions (region C and region D), which are defined analagously to regions A and B respectively but with another selection cut inverted.

In summary, we suppose we have the following input to our statistical analysis:

  • Observed data histogram in region D (hDataD)
  • Observed data histogram in region C (hDataC)
  • Observed data histogram in region B (hDataB)
  • Signal prediction histogram in region A (hSigA)
  • Irreducible background prediction histogram in region A (hBkgA)
  • Systematic variation of irreducible background prediction histogram in region A (hBkgA_syst)

We will also create a copy of hDataD and hDataB, called hDDBkgD and hDDBkgB, and use those as the nominal DD predictions in the regions C,D and A,B respectively.

(We will suppose we haven't yet looked at the data in the signal region, i.e. we are currently "blinded").

Workspace Building with xRooFit¶

Practical statistical analysis is done with workspaces, which are essentially a container object that can contain pdfs, datasets, and results of statistical analysis tasks such as fit results or hypothesis test scans. See here for more about workspaces.

We start by creating a workspace and wrapping it in an xRooNode to interact with it using the xRooFit API. Then we add a new empty multi-channel pdf (called simPdf conventionally).

Adding the histogram to the signal region caused a lot of things to happen:

  1. A new samples node was created in the signal region, which is a RooRealSumPdf that represents the sum of many samples
  2. A new factor of type SimpleDensity was created and added to the samples node.
    • This triggered the creation of a new regular observable (name given by the x-axis name from the histogram)
    • It also triggers creation of a prespecified parameter (called 1) that is used as the sample's coefficient (see here for more info)
  3. This new factor was scaled by a new factor of type Shape with parameters that will represent the mc-stat uncertainty
    • This triggered the creation of three floating parameters (prefixed by stat_), and three constant (prespecified) parameters (prefixed by tau_)
  4. The mc-stat uncertainty factors were constrained by adding Poisson constraint terms to the channel PDF
    • This triggered the creation of three global observables (prefixed by globs_)

Where the profile likelihood curve becomes equal to 1 can be used as an estimate of the 68% confidence interval on the parameter, i.e. the two values can be used as the $\pm 1\sigma$ uncertainties. These are known as the asymmetric errors or minos errors. The errors that are present in the fit result are from the diagonal elements of the covariance matrix. In the limit of large statistics the covariance matrix error (aka the symmetric error) will become equivalent to the asymmetric error.

We can get a sense of how different the errors are by testing what the value of the profile likelihood ratio is at the $\pm 1\sigma$ values given by the symmetric error:

In [29]:
graph = hs.graph("ts") # gets a TGraphErrors of the test-statistic (the profile likelihood ratio) of the scan
par_hat = fr.floatParsFinal().find("transferFactor")
print("Value of PLR @ best-fit + error = ",graph.Eval(par_hat.getVal()+par_hat.getError()))
print("Value of PLR @ best-fit - error = ",graph.Eval(par_hat.getVal()-par_hat.getError()))

Value of PLR @ best-fit + error =  0.9440681186876239
Value of PLR @ best-fit - error =  1.0874876643213742

We can evaluate minos errors on any parameter by setting its minos attribute before running a fit:

In [30]:
w.pars()["transferFactor"].setAttribute("minos")
fr2 = w["pdfs/simPdf"].reduced("CR*").nll("obsData").minimize() # rerun the fit
par_hat = fr2.floatParsFinal().find("transferFactor")
print("Value of PLR @ best-fit + error = ",graph.Eval(par_hat.getVal()+par_hat.getErrorHi()))
print("Value of PLR @ best-fit - error = ",graph.Eval(par_hat.getVal()+par_hat.getErrorLo()))

Value of PLR @ best-fit + error =  1.0127447211352414
Value of PLR @ best-fit - error =  1.004782812431617

From the above we see that the minos errors are indeed closer to where the PLL equals 1 (any difference will likely be due to at what precision the search algorithm runs for).
